1.数据库操作：①创建一个库，CREATE DATABASE 数据库名 ②创建一个基本表：CREATE TABLE <表名>(<列名><数据类型>[列级完整性约束条件][,<列名><数据类型>[列级完整性约束条件]]...[,<表级完整性约束条件>]) ③修改表结构—增加一个新列：ALTER TABLE 表名 ADD 列名 类型 ④修改表结构—修改一个新列：ALTER TABLE 表名 ALTER COLUMN 列名 新类型 ⑤修改表结构—删除一个列的语句：ALTER TABLE 表名 DROP COLUMN 列名  ⑥删除基本表：DROP TABLE 表名。
2.完整性约束：主键子句(Primary key:not null unique 不为空且唯一)、检查子句(check)、外键子句(Froeign key：关联其他的表)。
3.SQL的数据操纵语句：①：直接插入元组值：INSERT INTO 表名(列名序列) VALUES (元组值)
4.SQL访问控制：由DBA使用DBMS提供授权功能保证安全性。
①授权语句格式：GRANT<权限>[,<权限>]...[ON<对象类型><对象名>]TO<用户>[,<用户>]...[WITH GRANT OPTION】，PUBILC 接受权限的用户可以是单个或多个具体用户。WITH GRANT OPTIONS 若指定此句，则获得了权限的用户还可以将权限赋给其他用户。②收回权限语句格式：REVOKE<权限>[,<权限>]...[ON<对象类型><对象名>] FROM <用户>[,<用户>]...
5.视图： 视图是从一个或者多个基本表或视图中导出来的表，是虚拟表。
①视图的创建：CREATE VIEW 视图名(列表名) AS SELECT 查询子句 [WITH CHECK OPTION]
6.采用三级结构/两级映像的数据库体系  结构，如果对数据库的一张表创建聚簇索引，改变的是数据库的《内模式》。
7.函数依赖：如果X函数能决定Y或者Y依赖于X函数，记作X→Y；如果X→Y，但是X不包含Y，则称X→Y是非平凡函数依赖，一般都是非平凡函数依赖。                                                                                f
8.完全函数依赖：X中的所有的项全部联合起来才能决定Y，X→ Y
9.传递依赖：X→Y，X不包含Y，Y→Z，则Z对X传递依赖。
10.包含在候选关键字中的都是主属性
